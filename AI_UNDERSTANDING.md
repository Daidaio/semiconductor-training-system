# AI 智慧理解機制

## 🧠 雙層理解系統

系統現在使用**兩層理解機制**，讓你可以用最自然的方式說話！

### 第一層：規則式理解（快速）
- 使用關鍵字匹配
- 毫秒級回應
- 支援大量口語表達

### 第二層：AI 智慧理解（靈活）
- 當規則無法理解時自動啟動
- LLM 根據情境理解你的意圖
- 可以理解更自由、更模糊的表達

## 🎯 運作流程

```
你的輸入
   ↓
[第一層] 規則式 NLU
   ├─ 能理解？ → 直接執行 ✓
   └─ 不理解？ → 進入第二層
          ↓
     [第二層] AI 理解
          ├─ AI 能理解？ → 轉換成標準指令 → 執行 ✓
          └─ AI 也不確定？ → 請你說明確一點
```

## 📝 支援的自由表達類型

### 1. **非正式用語**
```
✅ 「那個過濾的東西」 → 過濾網
✅ 「溫度的部分」 → 溫度
✅ 「冷卻那邊」 → 冷卻系統
✅ 「光的強度」 → 光源強度
```

### 2. **模糊動作**
```
✅ 「看看那個」 → 檢查
✅ 「弄一下」 → 調整
✅ 「換掉」 → 更換
✅ 「關掉」 → 停機
```

### 3. **情境相關表達**
```
你： 先處理一下

你： 這個要怎麼辦
AI： [理解為] 詢問學長

你： 把這個弄好
AI： [根據問題] 你是想要「更換過濾網」嗎？
```

### 4. **問句形式**
```
✅ 「能不能看一下溫度」 → 檢查溫度
✅ 「要不要停機」 → 停機
✅ 「可以換嗎」 → 更換
✅ 「怎麼會這樣」 → 詢問學長
```

## 💡 使用技巧

### ✅ 好的習慣

1. **關鍵詞還是要有**
   ```
   ✅ 「那個冷卻的怎麼樣」（有「冷卻」）
   ❌ 「那個怎麼樣」（太模糊）
   ```

2. **情境相關的表達**
   ```
   ✅ 「先處理一下」（AI 會根據當前異常判斷）
   ✅ 「把這個弄好」（情境明確時可以）
   ```

3. **信任 AI 的理解**
   ```
   當 AI 問：「你是想要XXX嗎？」
   如果對：直接說「對」或「是」
   如果不對：說明確一點
   ```

### ⚠️ 要注意

1. **過於簡短**
   ```
   ❌ 「看」（看什麼？）
   ✅ 「看一下冷卻水」
   ✅ 「看溫度」
   ```

2. **完全沒線索**
   ```
   ❌ 「這個不行」（什麼不行？）
   ❌ 「有點怪」（哪裡怪？）
   ```

3. **多個動作混在一起**
   ```
   ❌ 「先看溫度然後調流量再換過濾網」
   ✅ 一次說一個動作
   ```

## 🚀 效能說明

### 規則理解（第一層）
- **速度**：< 1ms（幾乎即時）
- **準確度**：關鍵字匹配時 100%
- **支援範圍**：所有已定義的口語表達

### AI 理解（第二層）
- **速度**：
  - 本地 LLM：3-5 秒
  - Claude API：1-2 秒
- **準確度**：取決於表達清晰度
- **支援範圍**：幾乎所有自然表達

## 🎯 最佳實踐

1. **優先用第一層（規則）**
   - 使用常見口語表達
   - 回應最快

2. **讓 AI 幫忙理解**
   - 用自己最自然的方式說
   - AI 會幫你轉換

3. **配合 AI 的確認**
   - AI 不確定時會問你
   - 簡單確認「對」或「不對」

4. **越用越順**
   - 系統會學習你的表達習慣
   - 對話會越來越順暢

## 🎓 開始使用

```bash
python start_simulation.py
```
