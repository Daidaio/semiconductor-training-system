## 問題分析

### 原因
Gradio 無法直接載入相對路徑的本地圖片檔案。當使用 `<img src="interface/images/asml_duv.png">` 時，瀏覽器無法存取到該路徑。

### 解決方案
將圖片轉換為 **base64 編碼**，直接嵌入 HTML 中，這樣就不需要外部檔案路徑。

## 💡 技術細節（供參考）

### Base64 編碼原理：

```python
# 1. 讀取圖片
with open('asml_duv.png', 'rb') as f:
    image_data = f.read()

# 2. 轉換為 base64
image_base64 = base64.b64encode(image_data).decode('utf-8')

# 3. 構建 data URI
data_uri = f"data:image/png;base64,{image_base64}"

# 4. 嵌入 HTML
<img src="data:image/png;base64,iVBORw0KGgo...">
```

### 為什麼這樣可以？

瀏覽器支援 **Data URI Scheme**，允許將資料直接嵌入 URL：
```
data:[<mediatype>][;base64],<data>
```

例如：
```
data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...
```

這樣圖片就變成了一個「超長的文字字串」，不需要外部檔案！
