## 二、核心模組詳細架構

### 2.1 自然語言理解模組 (NLU Controller)

```
┌─────────────────────────────────────────────────────┐
│          Natural Language Controller                │
├─────────────────────────────────────────────────────┤
│                                                     │
│  使用者輸入                                          │
│      ↓                                              │
│  ┌──────────────────────────────────┐              │
│  │   雙層理解機制                    │              │
│  │                                  │              │
│  │  第一層：規則式匹配（優先）        │              │
│  │  ├─ 口語模式轉換                 │              │
│  │  ├─ 關鍵字識別                   │              │
│  │  ├─ 意圖分類                     │              │
│  │  └─ 參數提取                     │              │
│  │                                  │              │
│  │  ↓ (如果 intent == unknown)     │              │
│  │                                  │              │
│  │  第二層：AI語意理解（fallback）   │              │
│  │  ├─ LLM上下文理解                │              │
│  │  ├─ 模糊意圖推測                 │              │
│  │  └─ 標準化指令建議               │              │
│  └──────────────────────────────────┘              │
│      ↓                                              │
│  ┌──────────────────────────────────┐              │
│  │   解析結果                        │              │
│  │   • intent: check/ask/adjust...  │              │
│  │   • target: cooling/vacuum...    │              │
│  │   • params: {value, unit}        │              │
│  │   • confidence: 0.0-1.0          │              │
│  └──────────────────────────────────┘              │
│                                                     │
└─────────────────────────────────────────────────────┘

支援意圖類型：
- check: 檢查設備/參數
- ask: 詢問學長（靜默，只對話）
- adjust: 調整參數
- replace: 更換零件
- shutdown: 停機
- restart: 重啟
- calibrate: 校正
- clean: 清潔
```

## 三、資料流程圖

```
使用者輸入
    ↓
┌──────────────────┐
│  規則式NLU匹配   │ ← 口語模式庫、關鍵字庫
└──────────────────┘
    ↓
  已識別？
    ├─ 是 → 解析結果
    └─ 否 ↓
┌──────────────────┐
│   AI語意理解     │ ← 情境資訊、歷史對話
└──────────────────┘
    ↓
┌──────────────────┐
│   動作驗證       │ ← 當前狀態、約束規則
└──────────────────┘
    ↓
  是否為「詢問」？
    ├─ 是 → AI學長回答 → 對話歷史更新
    └─ 否 ↓
  是否為「檢查」？
    ├─ 是 → 執行檢查 → 設備狀態更新（靜默）
    └─ 否 ↓
┌──────────────────┐
│   動作執行       │ → 狀態更新
└──────────────────┘
    ↓
┌──────────────────┐
│   效果應用       │ → 情境演進
└──────────────────┘
    ↓
┌──────────────────┐
│  視覺更新        │ → 設備圖、儀表板、對話
└──────────────────┘
    ↓
┌──────────────────┐
│  AI動作回饋      │ → 對話歷史（可選）
└──────────────────┘
    ↓
  故障解決？
    ├─ 是 → 顯示完成訊息
    └─ 否 → 繼續演進（10秒後）
```

## 五、關鍵技術架構

### 5.1 雙層理解機制

```
用戶輸入: "請問學長，什麼情況會造成對準偏移"
    ↓
┌─────────────────────────────────┐
│  第一層：規則式快速匹配          │
│                                 │
│  正則模式:                       │
│  ^(請問學長|學長|專家).* → ask  │
│                                 │
│  結果: intent=ask ✓             │
│        confidence=1.0           │
└─────────────────────────────────┘
    ↓
  成功識別 → 直接使用
  (跳過第二層)


用戶輸入: "這個怎麼了" (模糊表達)
    ↓
┌─────────────────────────────────┐
│  第一層：規則式匹配              │
│  結果: intent=unknown ✗         │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  第二層：AI語意理解              │
│                                 │
│  輸入到LLM:                      │
│  - 用戶問題                      │
│  - 當前情境                      │
│  - 設備狀態                      │
│                                 │
│  LLM推測: "檢查溫度"            │
│                                 │
│  結果: intent=check             │
│        target=temperature       │
└─────────────────────────────────┘
```

### 5.2 靜默檢查機制

```
用戶輸入: "檢查冷卻水"
    ↓
解析: intent=check, target=cooling
    ↓
┌─────────────────────────────────┐
│  執行檢查                        │
│  • 讀取 cooling_flow = 3.5      │
│  • 讀取 filter_clogged = True   │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  更新設備狀態區塊                │
│                                 │
│  equipment_status = {           │
│    "冷卻水系統": {              │
│      "status": "warning",       │
│      "message": "流量: 3.5..."  │
│    }                            │
│  }                              │
└─────────────────────────────────┘
    ↓
❌ 不加入對話歷史
✅ 只更新視覺顯示
```

### 5.3 對話vs檢查分離

```
┌────────────────┬──────────────┬──────────────┐
│   輸入類型      │  對話歷史     │  設備狀態     │
├────────────────┼──────────────┼──────────────┤
│ "檢查冷卻水"    │  ✗ 不更新    │  ✓ 更新      │
│ (check)        │              │              │
├────────────────┼──────────────┼──────────────┤
│ "流量正常嗎"    │  ✓ 學長回答  │  ✗ 不更新    │
│ (ask)          │              │              │
├────────────────┼──────────────┼──────────────┤
│ "更換過濾網"    │  ✓ 系統訊息  │  ✓ 更新      │
│ (replace)      │              │              │
└────────────────┴──────────────┴──────────────┘
```
