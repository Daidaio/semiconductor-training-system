# Stage 2 實作訓練 - AI 學長整合指南

## 🎯 功能說明

現在 Stage 2 的實作訓練情境已經整合了 **AI 情境學長**！就像有個經驗豐富的學長在旁邊一起處理故障，對話更自然、更像真實現場。

## ✨ 新功能

### 1. **AI 學長式對話**
- ❌ 不再是機械式的模板回應
- ✅ 自然、輕鬆的學長口吻
- ✅ 會說「你看一下...」「我們先...」「試試...」
- ✅ 適時用「嗯」「對」「沒錯」等語助詞

### 2. **智慧引導**
- **觀察階段**：「你看到了什麼異常？」
- **分析階段**：「這些現象之間有什麼關聯？」
- **行動階段**：「現在最該做什麼？」
- **反問機制**：不直接給答案，引導你思考

### 3. **即時回饋**
- 執行關鍵動作時，學長會即時給予肯定
- 動作錯誤時，友善地提示換個方向
- 情況危急時，會直接提醒「這個要趕快處理」

### 4. **情境演進評論**
- 故障惡化時，學長會提醒注意
- 根據嚴重程度調整語氣
- 就像真的有人在旁邊看著情況變化

### 5. **AI 復盤**
- 情境結束後，學長會給總評
- 簡單評價表現（2-3句話）
- 指出做得好和可以改進的地方
- 分享關鍵經驗

## 🚀 使用方式

### 方法 1：使用本地 LLM（推薦，免費）

1. **安裝 Ollama**（如果還沒安裝）
   ```bash
   # Windows: 下載並安裝
   # https://ollama.com/download/windows

   # 安裝後，下載 Qwen 模型
   ollama pull qwen2.5:7b
   ```

2. **啟動訓練系統**
   ```bash
   cd semiconductor_training_system
   python start_simulation.py
   ```

3. **確認 AI 學長已啟動**
   看到這個訊息就成功了：
   ```
   [OK] 使用本地 LLM: qwen2.5:7b
   [OK] 使用 AI 情境學長模式
   ```

### 方法 2：使用 Claude API

1. **設定 API Key**
   ```bash
   # Windows PowerShell
   $env:ANTHROPIC_API_KEY="your-api-key-here"

   # Windows CMD
   set ANTHROPIC_API_KEY=your-api-key-here
   ```

2. **啟動系統**
   ```bash
   python start_simulation.py
   ```

### 方法 3：不使用 AI（模板模式）

如果 AI 不可用，系統會自動降級到模板回應模式，仍然可以正常訓練。

## 🎮 訓練流程

1. **開始情境**
   - 點擊「開始新情境」
   - 系統會生成隨機故障

2. **觀察與檢查**
   ```
   - 檢查冷卻水系統
   - 確認溫度感測器
   - 看看流量表
   ```

3. **詢問學長**（隨時可以）
   ```
   - 問專家為什麼會這樣
   - 請學長給建議
   - 這樣做對嗎
   ```

4. **採取行動**
   ```
   - 停機
   - 更換過濾網
   - 重新啟動
   ```

5. **完成復盤**
   - 查看評分和 AI 學長的總評
   - 學習關鍵知識點

## 🔧 進階設定

### 切換 AI 模型

編輯 `.env` 檔案（如果沒有就創建一個）：

```bash
# 本地 LLM 模型選擇
LOCAL_LLM_MODEL=qwen2.5:7b  # 或 qwen2.5:14b, qwen2.5:32b

# Claude API Key（選用）
ANTHROPIC_API_KEY=your-key-here
```

### 關閉 AI 學長模式

如果想使用傳統的專家顧問模式：

修改 `start_simulation.py`：
```python
demo = create_simulation_interface(secom_path, use_ai_mentor=False)
```

## 🐛 故障排除

### 問題：AI 回應很慢
**解決**：
- 本地 LLM 首次回應會慢一點（正常現象）
- 確認 Ollama 服務正在運行
- 考慮使用更小的模型（如 qwen2.5:7b）

### 問題：AI 學長沒有啟動
**解決**：
1. 檢查 Ollama 是否安裝並運行
2. 確認模型已下載：`ollama list`
3. 查看啟動訊息，確認使用哪個模式

### 問題：想使用 Claude 但一直用本地 LLM
**解決**：
- 本地 LLM 優先級最高
- 要用 Claude，需要停止 Ollama 服務
- 或在程式碼中強制指定
