# 3D 模型資源與測試指南

## 🛒 購買 3D 模型資源

### 推薦平台 1：TurboSquid（最專業）

#### 直接搜尋連結

```
https://www.turbosquid.com/Search/3D-Models?keywords=lithography+machine

https://www.turbosquid.com/Search/3D-Models?keywords=semiconductor+equipment

https://www.turbosquid.com/Search/3D-Models?keywords=wafer+stepper

https://www.turbosquid.com/Search/3D-Models?keywords=clean+room+equipment
```

#### 推薦模型（參考）

**方案 A：高品質模型** ($150-$299)
- 搜尋：「Industrial Equipment Semiconductor」
- 選擇：有 PBR 材質、高多邊形數、包含動畫
- 格式：確認支援 .glb 或 .fbx

**方案 B：中等品質模型** ($79-$149)
- 搜尋：「Manufacturing Equipment」
- 選擇：中等多邊形數、基本材質
- 適合：預算有限但需要真實感

**方案 C：基礎模型** ($29-$79)
- 搜尋：「Laboratory Equipment」
- 選擇：低多邊形、簡單材質
- 適合：快速原型驗證

#### 選購檢查清單

```
✅ 格式支援
  □ 支援 .glb 或 .gltf (最佳)
  □ 或支援 .fbx, .obj (可用 Blender 轉換)

✅ 材質貼圖
  □ 包含 Base Color (基礎顏色)
  □ 包含 Normal Map (法線貼圖)
  □ 包含 Roughness (粗糙度)
  □ 包含 Metallic (金屬度)

✅ 模型品質
  □ 多邊形數量: 20,000 - 100,000 (網頁適用)
  □ 零件分層命名清楚
  □ UV 展開正確
  □ 無破面或錯誤

✅ 授權許可
  □ 確認可用於學術/商業用途
  □ 確認可修改和整合
```

# -*- coding: utf-8 -*-
"""
測試 3D 真實模型載入
"""

import gradio as gr
from interface.equipment_visualizer_3d_realistic import RealisticEquipmentVisualizer

def test_3d_visualization():
    """測試 3D 視覺化"""

    # 初始化視覺化器
    visualizer = RealisticEquipmentVisualizer(
        model_path="assets/models/duv_machine.glb"
    )

    # 模擬設備狀態
    test_state = {
        'cooling_flow': 3.5,      # 異常（正常 5.0）
        'lens_temp': 25.2,        # 異常（正常 23.0）
        'vacuum_pressure': 2.1e-6,  # 警告（正常 1.0e-6）
        'light_intensity': 98.0,  # 正常
        'alignment_error': 8.5    # 異常（正常 < 2.0）
    }

    # 生成 3D 視圖
    html_output = visualizer.generate_3d_view(
        state=test_state,
        faults=['cooling_system', 'lens_system']
    )

    # 創建 Gradio 介面
    with gr.Blocks(title="3D 曝光機測試") as demo:
        gr.Markdown("# 🔬 DUV 曝光機 3D 視覺化測試")
        gr.Markdown("測試真實 3D 模型載入與異常告警效果")

        html_component = gr.HTML(html_output)

        gr.Markdown("""
        ## 🎮 互動說明
        - **滑鼠左鍵拖曳**：旋轉視角
        - **滑鼠滾輪**：縮放
        - **滑鼠右鍵拖曳**：平移
        - **點擊零件**：查看詳細資訊

        ## 🔴 告警狀態
        - 冷卻系統：**異常**（3.5 L/min，應為 5.0）
        - 鏡頭溫度：**異常**（25.2°C，應為 23.0°C）
        - 真空壓力：**警告**（2.1e-6 Torr）
        - 對準系統：**異常**（誤差 8.5 nm）

        異常零件會顯示**紅色閃爍光暈**
        """)

    return demo

if __name__ == "__main__":
    print("=" * 60)
    print("3D 真實模型測試")
    print("=" * 60)
    print()
    print("啟動測試介面...")
    print("開啟瀏覽器: http://127.0.0.1:7860")

    print("=" * 60)

        server_name="127.0.0.1",

        share=False
    )

## 🔧 疑難排解

### 問題 1：模型載入失敗

**解決方案**：

1. 確認檔案路徑
```bash
# 檢查檔案是否存在
ls assets/models/duv_machine.glb
```

2. 確認檔案格式
```bash
# 檔案大小應該 > 1MB
ls -lh assets/models/duv_machine.glb
```

3. 嘗試用 Blender 重新導出
   - 開啟 Blender
   - File → Import → 你的原始格式
   - File → Export → glTF 2.0 (.glb)
   - Format: glTF Binary (.glb)
   - 確認勾選 Compression

### 問題 2：模型太大，載入很慢

**解決方案**：

用 Blender 優化模型

```
1. 開啟模型
2. 選擇所有物件 (A 鍵)
3. 添加 Decimate Modifier
   • Ratio: 0.5 (減少一半多邊形)
4. Apply Modifier
5. 重新導出 .glb
```

### 問題 3：找不到零件，無法添加告警

**原因**：模型中的零件名稱與程式碼不匹配

**解決方案**：

修改 `equipment_visualizer_3d_realistic.py` 中的 `component_names`

```python
self.component_names = {
    'lens_system': ['你的模型中的鏡頭名稱', 'Lens', 'ProjectionLens'],
    'wafer_stage': ['你的模型中的載台名稱', 'Stage', 'Chuck'],
    # ... 根據實際模型調整
}
```

**如何查看模型中的零件名稱**：

1. 用 Blender 開啟 .glb 檔案
2. 查看 Outliner 視窗（右上角）
3. 記錄每個 mesh 的名稱
4. 更新 `component_names` 映射

### 問題 4：模型顯示是黑色的

**原因**：材質或光源問題

**解決方案**：

1. 確認模型有材質
   - 用 Blender 開啟
   - 切換到 Shading 工作區
   - 確認每個物件都有材質

2. 調整程式碼中的光源亮度
```python
# 在 addLights() 函數中
const ambientLight = new THREE.AmbientLight(0x404565, 2.5); // 增加亮度
```

## 📝 整合到訓練系統

### 修改 `simulation_interface.py`

```python
# 在 __init__ 方法中
from interface.equipment_visualizer_3d_realistic import RealisticEquipmentVisualizer

# 添加 3D 視覺化器
self.equipment_visualizer_3d = RealisticEquipmentVisualizer(
    model_path="assets/models/duv_machine.glb"
)
print("[OK] 3D 真實模型視覺化器已載入")

# 在介面中使用
def get_3d_equipment_view(self):
    """獲取 3D 設備視圖"""
    current_state = self.digital_twin.get_current_state()
    faults = self._detect_faults(current_state)

    return self.equipment_visualizer_3d.generate_3d_view(
        state=current_state,
        faults=faults
    )
```
