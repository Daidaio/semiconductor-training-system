# 性能優化報告

## 問題一：AI 回應速度優化 ✅ 已解決

### 原始問題
用戶反應：「他回應太慢，導致跑不出來」

### 問題分析
經檢查程式碼，發現三個主要效能瓶頸：

1. **Timeout 設定過長**
   - 原設定：60 秒
   - 問題：即使 LLM 反應慢，也會一直等待

2. **System Prompt 過長**
   - 原長度：約 900+ 字的詳細角色設定
   - 問題：LLM 每次都要處理大量系統提示詞

3. **Context 資訊過多**
   - 原做法：包含完整情境、動作歷史、所有狀態參數
   - 問題：輸入 token 數過多，拖慢回應

4. **無輸出長度限制**
   - 問題：LLM 可能生成很長的回應

### 優化方案

#### 1. 降低 Timeout（[local_mentor_bot.py:166](stage1_theory/local_mentor_bot.py#L166)）
```python
# 修改前
timeout=60  # 60 秒

# 修改後
timeout=15  # 降低到 15 秒
```

#### 2. 限制輸出長度（[local_mentor_bot.py:163](stage1_theory/local_mentor_bot.py#L163)）
```python
"options": {
    "temperature": 0.7,
    "top_p": 0.9,
    "top_k": 40,
    "num_predict": 150,  # 新增：限制最大輸出約 150 tokens
}
```

#### 3. 精簡 System Prompt（[local_mentor_bot.py:50-60](stage1_theory/local_mentor_bot.py#L50-L60)）
```python
# 修改前：900+ 字的詳細說明
"""你是一位經驗豐富的半導體設備工程師學長...
[大量角色設定、對話風格範例、教學方法說明]
"""

# 修改後：精簡到 ~150 字
"""你是半導體設備工程師學長，指導學弟。

風格：
- 親切自然，像學長帶學弟
- 用「欸」「啦」「你看」等口語
- 簡短回答（2-3句）+ 反問確認理解

記住：簡短、口語、反問。"""
```

#### 4. 簡化 Context 建構（[ai_scenario_mentor.py:167-181](core/ai_scenario_mentor.py#L167-L181)）
```python

# 修改後：精簡到一行
context = f"[{stage_name}] 溫度{temp}°C,流量{flow}L/min"

# 問題也從多行變一行
full_question = f"{context} 學弟問：{question}\n回應2-3句，用學長口吻。"
```

#### 5. 精簡情境系統提示（[ai_scenario_mentor.py:89-97](core/ai_scenario_mentor.py#L89-L97)）
```python
# 從 ~400 字精簡到 ~80 字
scenario_system_prompt = """你是半導體設備現場學長，協助學弟處理故障。

風格：
- 輕鬆自然，像學長帶學弟
- 用「你看」「試試」等口語
- 引導思考，不直接給答案
- 簡短回應（2-3句）

記住：自然、實用、簡短。"""
```

#### 方案 B：基於規則的預測系統 ⭐⭐⭐⭐

**做法**：
定義一組規則來預測即將發生的故障

```python
class RuleBasedPredictor:
    def predict(self, state_history: List[Dict]) -> Dict:
        """基於規則的預測"""

        # 規則 1：溫度持續上升
        if self._is_temperature_rising(state_history):
            return {
                "warning": "溫度持續上升趨勢",
                "risk_level": "中",
                "suggested_action": "檢查冷卻系統"
            }

        # 規則 2：流量逐漸下降
        if self._is_flow_decreasing(state_history):
            return {
                "warning": "冷卻流量下降趨勢",
                "risk_level": "高",
                "suggested_action": "檢查過濾網和管路"
            }

        # 規則 3：真空壓力波動
        if self._is_pressure_unstable(state_history):
            return {
                "warning": "真空壓力不穩定",
                "risk_level": "高",
                "suggested_action": "檢查密封件"
            }
```

**整合到介面**：

**優點**：

#### 方案 C：混合式預測系統 ⭐⭐⭐⭐⭐

**做法**：結合 ML 模型 + 規則系統

```python
class HybridPredictor:
    def __init__(self):
        self.ml_model = load_trained_model()
        self.rule_engine = RuleBasedPredictor()

    def predict(self, current_state, state_history):
        # 1. ML 模型給出機率預測
        ml_result = self.ml_model.predict(current_state)

        # 2. 規則引擎給出趨勢判斷
        rule_result = self.rule_engine.predict(state_history)

        # 3. 綜合兩者結果
        if ml_result["probability"] > 0.7 and rule_result["risk_level"] == "高":
            return {
                "alert": "高風險！ML模型與規則都檢測到異常",
                "confidence": "高",
                "action": "建議立即檢查"
            }
```

**優點**：
- ✅ 結合兩者優點
- ✅ 更可靠
- ✅ 適合實際產線應用

**實作難度**：⭐⭐⭐⭐（較高）

### 實作範例（方案 B - 快速版）

```python
# 在 scenario_engine.py 加入預測模組

class FaultPredictor:
    """簡易故障預測器"""

    def predict_fault(self, state_history: List[Dict]) -> Optional[Dict]:
        """
        分析狀態歷史，預測可能的故障

        Args:
            state_history: 最近 10 筆狀態記錄

        Returns:
            預測結果或 None
        """
        if len(state_history) < 5:
            return None

        recent_5 = state_history[-5:]

        # 檢查溫度趨勢
        temps = [s.get("lens_temp", 0) for s in recent_5]
        if self._is_rising_trend(temps) and temps[-1] > 24.5:
            return {
                "type": "temperature",
                "message": "溫度持續上升，預計 2-3 分鐘後可能超過警戒值",
                "probability": 0.75,
                "suggested_action": "建議檢查冷卻系統"
            }

        # 檢查流量趨勢
        flows = [s.get("cooling_flow", 0) for s in recent_5]
        if self._is_falling_trend(flows) and flows[-1] < 4.8:
            return {
                "type": "cooling",
                "message": "冷卻流量持續下降，可能即將發生異常",
                "probability": 0.80,
                "suggested_action": "建議檢查過濾網"
            }

        return None

    def _is_rising_trend(self, values: List[float]) -> bool:
        """判斷是否為上升趨勢"""
        if len(values) < 3:
            return False
        return values[-1] > values[-2] > values[-3]

    def _is_falling_trend(self, values: List[float]) -> bool:
        """判斷是否為下降趨勢"""
        if len(values) < 3:
            return False
        return values[-1] < values[-2] < values[-3]
```
