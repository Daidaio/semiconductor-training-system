## 修改重點

### 之前的問題
- 使用 `Textbox` 顯示訊息
- 訊息用字串拼接累積
- 有 `max_lines=20` 限制
- **對話歷史會被刷新掉**

### 現在的解決方案
- 改用 `Chatbot` 組件
- 訊息用列表管理
- 無行數限制
- **完整保留所有對話歷史**

## 技術細節

### 資料結構改變

**之前（字串拼接）**：
```python
# 容易出錯，可能被覆蓋
system_message = f"{system_message}\n\n[你] {input}\n\n{response}"
```

**現在（列表管理）**：
```python
# 確保歷史保留
self.conversation_history.append((user_input, system_response))
```

### Gradio 組件改變

**之前**：
```python
system_messages = gr.Textbox(
    label="系統訊息",
    lines=15,
    max_lines=20,  # 限制高度
    interactive=False
)
```

**現在**：
```python
system_messages = gr.Chatbot(
    label="對話歷史",
    height=400,  # 固定高度，內部可滾動
    show_label=True
)
```

## 與之前的差異

使用方式**完全相同**，唯一差異是：

| 項目 | 修改前 | 修改後 |
|------|--------|--------|
| 顯示方式 | Textbox | Chatbot（聊天氣泡） |
| 對話歷史 | ❌ 容易刷新 | ✅ 完整保留 |
| 顯示長度 | 最多 20 行 | 無限制（可滾動） |
| 視覺效果 | 純文字 | 對話氣泡，更清晰 |
| 回看歷史 | ❌ 困難 | ✅ 可以往上滾動 |

## 其他優化

同時也優化了：

1. **對話格式**：不再需要手動加「========」分隔線
2. **用戶/系統分離**：清楚看到誰說了什麼
