```
  ├─ sensor_readings = sensors.read_all() (讀取即時數據)

  ↓

  ├─ 每 1 秒監控一次

  • 警報訊息
```

### 2. 即時監控循環 (每 1 秒)

```
[背景執行緒運行]
  ↓
sensors.read_all()
  ├─ cooling_flow.read() → 3.5 L/min (固定故障值)
  ├─ lens_temp.read()    → 23.01°C (逐秒上升 +0.01) ⭐
  ├─ vacuum_pressure.read() → 1.0e-6 Torr
  ├─ ... (其他 5 個感測器)
  ↓
ProcessParameterDB.check_parameter_status()
  ├─ cooling_flow: 3.5 vs 5.0 → 偏移 30% → CRITICAL ❌
  ├─ lens_temp: 23.01 vs 23.0 → 偏移 0.04% → NORMAL ✓
  ↓
ClosedLoopController._check_parameter()
  ├─ if CRITICAL and not in active_alarms:
  │    ├─ _trigger_alarm()  ⭐
  │    ├─ alarm_id = "ALARM-0001"
  │    ├─ diagnosis = "冷卻水流量過低 (3.5 < 5.0)"
  │    ├─ suggestion = "1) 檢查閥門\n2) 檢查過濾網\n..."
  │    └─ alarm_callback(alarm) → 輸出到後台
  ↓
[1秒後繼續監控]
  lens_temp: 23.01 → 23.02°C ... → 23.10°C
  ↓
  if lens_temp > 23.1°C:
    _trigger_alarm() → ALARM-0002 (WARNING)
```

```

  ↓
[數秒後]
  ├─ lens_temp: 23.10 → 23.09 → ... → 23.01 → 23.00°C
  ├─ closed_loop._clear_alarm("lens_temp")
  └─ handling_duration = 1.0 秒
  ↓
所有告警解除,紅燈消失 ✓
```

### 可以在論文中強調的核心貢獻

#### 1. 完整數位孿生架構實現

> 本研究基於 Liu et al. (2024) 的五維數位孿生架構,實現了完整的半導體設備教學訓練系統。系統包含:
>
> - **實體層**: 8 個高擬真模擬感測器,基於 SECOM 真實數據統計特性
> - **資料層**: 製程參數資料庫 (8 參數) + SECOM 資料集 (1,567 樣本)
> - **虛擬層**: 真實設備照片視覺化 + 即時狀態映射
> - **應用層**: Gradio Web 介面 + AI 導師 + 閉環控制系統
> - **連接層**: Python 模擬通訊協定 (每秒採集)

#### 2. 閉環控制機制創新

> 相較於論文的人工介入式閉環控制,本系統實現了**智能化閉環控制**:
